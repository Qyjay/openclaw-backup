一、文档概述

1.1 文档说明

本文档为「本产品」产品需求文档（PRD），完整阐述产品定位、目标用户、需求分析、技术架构、功能设计、迭代规划与商业构想。

「本产品」是一款面向大学生的 AI 生活伙伴，以零门槛生活记录为入口，融合**AI 内容创作**、AI 学习陪伴与智能体社交功能，构建"记录 → 创作 → 理解 → 陪伴 → 连接"的完整闭环。区别于传统日记工具的"工具思维"和传统社交平台的"流量思维"，「本产品」探索的是Agent 思维——每个用户拥有一个持久化的 AI Agent，它记得你的一切、主动关心你、帮你管理生活、替你完成社交前置筛选，还把你的日记变成漫画、小说和可分享的艺术作品。

本产品的技术内核借鉴了 OpenClaw 开源 Agent 框架的核心理念：持久化记忆、主动式行为、工具编排、多 Agent 协作，并首次将 Agent to Agent（A2A） 协议应用于校园社交场景。

1.2 产品定位

一句话定义：你的大学 AI 伙伴——记录生活、创作内容、精准社交、助力成长。

「本产品」不是一个日记 App，不是一个社交 App，也不是一个学习 App——它是你大学四年的 AI 同行者。

它在你身边：

- 你拍一张照，它帮你写日记
- 你记够 5 条，它把你的一天画成漫画
- 你说一句"好累"，它知道是因为什么，给出真正有用的安慰
- 你想找人吃饭，它已经在后台帮你找好了搭子
- 你大四写简历，它从四年日记中自动提取你的成长素材
- 你毕业时，收获一部属于你的大学自传小说

4.2 高频核心场景

场景一：随手记录（日活核心）

触发：用户随手拍了一张/多张照片
频率：每天 2-5 次
流程：
  ① 用户从相册选择或即时拍摄 1-N 张照片
  ② 可选：添加一句文字或短语音（也可以什么都不加）
  ③ AI 自动分析照片内容、场景、时间、地点、天气
  ④ 融合用户历史写作风格，生成个性化日记
  ⑤ 用户可编辑/调整风格（文艺/搞笑/简洁/吐槽体）
  ⑥ 保存 → 自动归入知识库 → 更新用户画像

技术链路：
  [照片] → Vision API（场景/物体/人物/情绪识别）
         → LBS API（获取地点）
         → Weather API（获取天气）
         → RAG（检索历史日记风格+画像）
         → LLM（融合所有上下文 → 生成日记）
         → Function Calling（提取标签 → 更新画像）

场景二：找搭子（社交核心）

触发：用户想找人一起做某件事
频率：每周 2-3 次
流程：
  ① 用户输入需求："想找人周六去天津之眼拍照"
  ② 用户的 Agent 解析需求 → 结构化：
     {类型: 旅行搭子, 时间: 周六, 地点: 天津之眼, 兴趣: 摄影}
  ③ Agent 在后台与其他用户的 Agent 进行 A2A 匹配
  ④ 匹配逻辑：时间空闲 + 兴趣匹配 + 地理可达 + 社交偏好兼容
  ⑤ 找到候选 → Agent 间预沟通 → 确认匹配度
  ⑥ 推送给双方用户："找到一个匹配！共同点：都喜欢摄影，都想去天津之眼"
  ⑦ 双方确认 → 建联（站内聊天或跳转微信）
  ⑧ 活动后：上传照片 → 日记自动记录 → 匹配反馈 → 优化后续推荐

A2A 协议：
  Agent A → 匹配引擎：发布需求
  匹配引擎 → 候选 Agent B/C/D：询问意愿
  Agent B → 匹配引擎：回复画像摘要 + 时间确认
  匹配引擎 → Agent A：返回匹配结果
  Agent A → 用户 A：推送建联建议

场景三：AI 陪伴对话（情感核心）

触发：用户主动找 AI 聊天，或 AI 检测到情绪异常主动关怀
频率：每天 1-3 次
流程：
  ① 用户："今天好焦虑啊"
  ② AI 检索近期日记，定位焦虑来源
  ③ AI 回复（基于对用户的深度了解）：
     "是因为雅思吗？还有 43 天，你的阅读正确率最近一直在涨，
      上周从 62% 提到了 72%。按这个趋势，考到目标分数没问题的。
      要不先做一个番茄钟？做完心情会好一点。"
  ④ 这不是通用 AI 安慰，而是基于用户真实数据的精准陪伴

区别于传统 AI 对话：
  ❌ ChatGPT："我理解你的焦虑，建议你深呼吸..."（通用话术）
  ✅ 产品 AI："你的雅思阅读上周从 62% 提到 72%..."（真正了解你）

把一篇日记自动转化为 4-6 格漫画，支持多种漫画风格。这是**最大的差异化亮点**——市面上没有任何日记 App 做到这一点。

🇺🇸 美漫 | 硬朗线条、浓重阴影、超级英雄感
📱 条漫 | 竖版长条、简笔画、表情包风

生成流程：
```
日记文本
  → LLM 拆分为 4-6 个关键场景描述
  → 每个场景生成图片 prompt（含固定角色描述保证一致性）
  → AI 图片生成 API 批量生成
  → 前端拼接为漫画排版
  → 叠加对话气泡和旁白文字
```

解锁条件（游戏化激励）：
- 当天碎片记录 ≥ 5 条 → 解锁当日漫画生成
- 连续记录 7 天 → 永久解锁本周所有日记的漫画
- 每周限免 1 次（降低尝鲜门槛）

角色一致性方案：
- 用户首次使用时，AI 引导用户选择/描述自己的漫画形象（性别、发型、穿着风格）
- 保存为角色 prompt 模板（如"a male college student with short black hair, wearing a blue hoodie"）
- 每次生成漫画时，所有场景复用同一角色描述

- **LLM 维护角色表**：从日记中提取出现的人物（室友/老师/搭子），赋予小说化名和性格描写
- **保持连续性**：每次生成新章节时，带入前几章摘要作为上下文（类似 Agent 持久记忆）

- 支持多种声音风格（温暖男声/治愈女声/播音腔/二次元等）
- 可生成"今日回顾播客"——把当天所有日记串联成一期 5-10 分钟的个人播客
- 支持边走边听（后台播放）

（6）日记 BGM 生成

根据日记情绪自动配背景音乐，让每篇日记都有专属"氛围感"。

- AI 分析日记情绪 → 自动匹配音乐风格（欢快/舒缓/忧伤/热血）
- 调用 MiniMax 音乐生成 API 生成 30 秒 BGM
- 日记 + BGM + 照片 = 完整的"日记 MV"
- 可用于分享卡片的背景音乐

（7）漫画周刊 / 学期 MV

- **漫画周刊**：每周自动把 7 天的漫画编排成一期"个人周刊"，像漫画杂志一样有封面、目录、章节
- **学期 MV**：学期结束时，自动剪辑日记照片 + BGM，生成 1-2 分钟的回忆视频（调用 MiniMax 视频生成 API）

（8）时光胶囊 / 给未来的信

- **时光胶囊**：写给指定未来日期的自己的信，到期自动推送
- **"去年今天"对比**：日记 vs 日记、漫画 vs 漫画的时间维度对比
- **成长可视化**：基于日记的 AI 性格分析按月追踪，展示个人成长轨迹

6.1.4 创作引擎的增长飞轮

```
用户记录碎片 → 解锁文风日记（门槛低，成就感高）
  → 解锁漫画生成（惊喜感，分享冲动）
    → 分享到社交平台（免费品牌曝光）
      → 朋友看到"别人的漫画日记" → 下载体验
        → 新用户开始记录 → 循环
```

同时，大学自传小说功能创造了**长期留存壁垒**——用户的小说在持续更新，四年的故事越来越完整，迁移成本极高。

七、隐私安全设计

7.1 数据分层保护

A2A 社交的核心顾虑是隐私。「本产品」采用三层数据分级：

┌─────────────────────────────────────────────────┐
│  Layer 1: 可共享层（A2A 匹配时使用）             │
│  ─────────────────────────────────────           │
│  兴趣标签、时间偏好、学院（可选）、社交需求       │
│  → 仅传输脱敏标签，不传输原始内容                │
│  → 用户可逐项控制哪些标签可共享                  │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│  Layer 2: 本地分析层（仅设备端处理）             │
│  ─────────────────────────────────────           │
│  日记全文、情绪记录、具体行踪、人际关系          │
│  → AI 分析在本地完成，结果以标签形式上传          │
│  → 复赛可用蓝心端侧模型实现本地推理              │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│  Layer 3: 绝对隐私层（永不离开设备）             │
│  ─────────────────────────────────────           │
│  真实姓名、手机号、身份信息、财务信息             │
│  → 加密存储，任何情况下不上传                    │
└─────────────────────────────────────────────────┘

一、文档概述
1.1 文档说明
本文档为个人陪伴与社交助手PRD，用于明确产品定位、拆解用户需求、规划核心功能模块、梳理实现逻辑与业务闭环、制定分阶段迭代方案，同时为产品研发、跨团队协作、项目评审、版本管控提供标准与依据。
本产品聚焦智能生活日记、智能体社交两个核心功能，以零门槛随手记录为流量入口，持续收集用户生活数据搭建专属个人智能体，优先实现AI智能生活日记基础功能，最终实现智能体社交功能，实现记录-社交-反哺记录的闭环。
1.2 产品定位
本产品是一款AI个人生活记录、陪伴以及智能社交工具，以零门槛生活记录为入口，融合心理陪伴与智能体社交功能，通过AI技术实现日常记录👉生活日记（多媒体生活日记/生活手账？）生成👉个人智能体搭建👉智能体分身社交👉现实建联闭环，打造专属个人知识库与数字分身，形成个人数字生活资产，降低用户生活记录与社交的成本，实现高效精准社交、虚拟互动，兼顾实用性、情感价值与社交创新性。
1.3 目标用户
核心用户：
- 不爱手动记日记、习惯随手拍照记录生活的全年龄段用户
- 有精准交友、找搭子、高效社交需求，排斥无效社交的年轻群体
1.4 核心价值
1. 降低记录门槛：用户可通过拍照、上传图片，并对图片进行局部编辑，根据个人喜好添加一句文字或简短语音记录，即可生成个人生活日记，解决用户“不想写、没时间写”的痛点
2. 打造数字资产：收集用户个人生活数据，形成私有知识库，孵化专属个人智能体分身
3. 全场景陪伴：生活记录，覆盖日常、自我成长两大需求
4. 创新社交模式：智能体社交，替代用户完成前置筛选，实现精准高效建联
二、产品需求分析
2.1 市场背景与行业痛点
当前市面上的记录类工具与社交平台大多相互割裂。传统日记APP依赖用户手动码字进行记录，耗费用户时间精力，使得用户留存较少；传统社交平台依赖用户主动刷选、自我介绍，存在大量无效社交、信息不对称等问题。
同时，年轻群体普遍拥有碎片化记录需求、通过简单的文字和图片难以整理个人生活、渴望精准交友/找搭子等痛点，市场上暂无一款整合轻量记录、智能体社交的一体化工具，存在创新空间。
2.2 用户核心需求
2.2.1 生活记录需求
- 行动门槛高：大多数人无手写or打字记日记的习惯，觉得耗时、繁琐，难以长期坚持
- 整理成本高：随手拍的照片散落在相册，无法串联成完整个人生活故事
- 形式单一：纯文字记录缺乏趣味性，无法满足音视频等多元化记录需求
2.2.2 社交需求
- 无效社交泛滥：传统社交平台匹配机制粗糙，难以找到兴趣、需求同频的人
- 社恐社交障碍：主动打招呼、自我介绍的社交模式，让社恐用户望而却步
- 需求匹配低效：找饭搭子、学习搭子、出游伙伴时，信息对接慢、筛选成本高
- 社交人设割裂：现实性格与社交平台表达不一致，难以遇到真正同频的人
2.3 用户使用场景分析
2.3.1 高频场景
1. 外出游玩/日常通勤随手拍照，不想打字，想快速生成当日生活日记
2. 翻看过往照片，想回忆当时的心情或进行个人生活总结
3. 想找学习/干饭/出游搭子，不想主动社交，希望AI进行即时精准筛选
2.3.2 延伸场景
1. 复盘一周/一月生活，通过AI总结个人生活轨迹与情绪变化
2. 接入智能设备数据，让健康数据（睡眠、心率）融入生活记录，丰富智能体画像
2.4 竞品简析与差异化优势（待补充）
市面上同类产品（日记类、传统社交平台）均存在功能单一、门槛高的问题：普通日记APP无AI生成能力，社交平台依赖人工筛选。
本产品差异化优势在于：以零门槛拍照记录为切口，收集用户数据孵化专属智能体分身，智能体社交模式，替代用户完成前置社交筛选，实现精准且高效建联，形成记录-分身-社交-记录的闭环
三、核心功能模块规划（优先级排序）
结合需求分析、用户痛点与场景，功能落地顺序：AI智能生活日记👉智能体社交，两个功能数据互通、相互赋能。
3.1 核心模块一：AI智能生活日记
3.1.1 功能定位
零门槛生活记录工具，也是智能体的数据喂养入口，以拍照（图片上传）为核心入口，通过AI技术自动生成日记、拓展音视频创作，收集个人全维度数据，从而搭建个人专属知识库，为智能体分身打造提供支撑，包括记录、编辑、复盘、数据资产化等功能实现。
3.1.2 核心子功能
（1）拍照上传与图片编辑
- 基础操作：支持单张/多张照片随手上传，适配手机相册上传、即时拍照（？）两种模式；
- 图片编辑：集成轻量化图片编辑功能（基于手绘指令的局部图片修改、图片融合），解决用户拍照不满意、需要微调的痛点
（2）AI自动生成日记
- 实现逻辑：基于RAG技术+大模型能力，解析图片内容、场景、（时间、地点）、情绪等信息，自动生成文字日记，无需用户手动输入大量内容；
- 生成效果：贴合用户日常语气，还原生活场景，支持单张照片生成短句日记、多张照片串联生成完整生活故事；
- 拓展输入：后续可接入语音、智能设备数据（如Apple Watch睡眠、心率数据），丰富日记生成维度与智能体用户画像。
（3）AI音视频创作
- AI生成音乐：调用第三方音乐生成API，通过大模型提示词匹配图片氛围，自动生成专属背景音乐，支持风格切换；
- AI生成动态视频：调用视频生成模型，将静态图片转化为动态短视频，实现图片“动起来”的效果，操作极简，一键生成；
（4）个人知识库与智能体搭建
- 数据收集：所有生成的日记、原始图片、音视频内容、用户兴趣偏好，自动归集为用户私有知识库，支持分类、检索、复盘；
- Agent训练：将日记全量数据作为AI陪伴助手的训练素材，让助手深度复刻用户性格、喜好、生活习惯、社交需求，打造高度贴合本人的数字分身。
3.1.3 业务闭环
用户随手拍照👉AI自动生成日记/音视频👉内容收集用于个人知识库👉知识库喂养个人智能体👉智能体参与社交筛选👉社交结果激发用户更多记录行为，反哺智能体成长。
3.2 核心模块三：智能体社交
3.2.1 功能定位
智能社交板块，采用A2A（Agent to Agent）模式，基于用户生活日记功能收集的个人数据，打造专属数字分身，替代用户在后台完成社交筛选、兴趣匹配，实现模拟人生互动，解决无效社交、社恐障碍、需求匹配低效等痛点，实现高效精准的现实建联。用户还可以在此板块输入即时需求，如寻找吃饭搭子、旅游搭子等，通过智能体即时寻找并匹配时间地点符合、兴趣一致的用户。
3.2.2 核心子功能
（1）个人智能体创建
- 分身生成：用户授权后，系统自动抓取个人日记、情绪、兴趣、生活轨迹等数据，逐渐生成高度复刻本人的AI智能体，作为用户数字分身；
- 人设校准：支持用户手动微调智能体标签（性格、兴趣、社交需求、择偶/交友偏好）
- 数据权限：智能体仅调用用户授权的日记数据，严禁未经授权的信息泄露，保障隐私安全。
（2）智能体后台社交与筛选
- 无感社交：用户上课、睡觉、工作期间，个人智能体在后台自动运行，与其他用户智能体进行数据交互、信息交换；
- 精准匹配：智能体基于兴趣爱好、生活习惯、社交需求（交友、恋爱、找搭子），自动筛选同频对象，判断对方是否符合用户预期；
- 需求匹配：针对搭子需求（学习、干饭、出游、运动），智能体快速对接有相同诉求的其他智能体，完成前置筛选；
- 建联通知：筛选到合适对象后，智能体向用户发送推送通知，展示匹配结果，用户可选择同意后，跳转微信等社交平台完成现实建联（or软件内部实现好友功能？）。
（3）社交闭环与数据反哺
- 现实建联成功后，用户可将社交过程、新好友互动记录同步至日记，进一步丰富个人知识库
- 每一次社交匹配、建联结果，都会优化智能体的筛选逻辑，让后续匹配更精准
- 优质匹配体验激发用户分享意愿，带动新用户注册，扩大智能体社交池，逐步形成社交生态
四、非功能需求
4.1 技术需求
- 支撑图片、音视频、日记数据、智能体模型的存储与算力管理
- 接入LLM API、RAG检索、MLLM API、智能体交互模型
- 用户隐私数据加密存储，智能体交互数据脱敏，完善权限管控机制
4.2 性能需求
- 图片上传、AI生成日记/音视频响应速度快，用户等待时间不宜过长
- 智能体后台运行低功耗，不占用手机过多资源
- 界面简洁易操作，符合移动端使用习惯，降低用户学习成本
