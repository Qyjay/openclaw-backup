**三层导航体系**：

| 层级 | 组件 | 触发方式 | 内容 |
|---|---|---|---|
| L1 侧边栏 | Drawer（左侧抽屉） | 首页左上角头像点击 / 右滑手势 | 用户信息 + 快捷入口 + 设置 |
| L2 TabBar | 底部导航栏（5项） | 点击切换 | 日记/发现/写/消息/我的 |
| L3 页面栈 | 二级页面 Stack | 页面内跳转（push） | 日记详情/番茄钟/设置等 |

## 1.2 侧边栏（Drawer）设计

**触发**：日记首页左上角用户头像 / 从屏幕左边缘右滑 ≥ 40rpx
**关闭**：点击右侧遮罩 / 左滑 / 点击侧边栏内的任何导航项
**宽度**：屏幕宽度 × 0.75（约 292rpx @ 390pt 设备）
**遮罩**：`rgba(0,0,0,0.4)`，点击关闭
**动画**：`transform: translateX()` 300ms ease-out

## 1.3 TabBar 设计

**位置**：屏幕底部固定
**高度**：98rpx（含安全区 padding-bottom）
**背景**：`#FFFFFF` + 顶部 `box-shadow: 0 -1rpx 0 rgba(0,0,0,0.06)`

| 位置 | 图标 | 文字 | 路由 | 说明 |
|---|---|---|---|---|
| 1 | 📔 diary | 日记 | /pages/index/index | 日记时间线（首页） |
| 2 | 🧭 discover | 发现 | /pages/discover/index | 功能入口网格 |
| 3（中间） | ✏️ write | — | /pages/write/index | **凸起按钮**，无文字 |
| 4 | 💬 message | 消息 | /pages/messages/index | AI对话 + 消息 |
| 5 | 👤 profile | 我的 | /pages/profile/index | 个人中心 |

**中间"写"按钮特殊处理**：
- 尺寸：96rpx × 96rpx 圆形，向上凸出 TabBar 20rpx
- 背景：主色渐变 `linear-gradient(135deg, #E8855A, #F0A882)`
- 图标：白色羽毛笔 icon，36rpx
- 阴影：`0 4rpx 16rpx rgba(232, 133, 90, 0.35)`
- 点击：弹出写日记 ActionSheet（拍照/相册/文字/语音），不直接跳页面

**写按钮 ActionSheet 弹层**：

## 1.4 顶部导航栏（NavBar）

一级页面使用自定义 NavBar，二级页面使用带返回箭头的标准 NavBar。

**一级页面 NavBar**：
```
┌──────────────────────────────────────┐
│  [头像]   页面标题          [功能图标] │
└──────────────────────────────────────┘
```
- 左侧：用户头像（仅日记首页，点击打开侧边栏）/ 无（其他页）
- 中间：页面标题
- 右侧：搜索图标（日记页）/ 无

**二级页面 NavBar**：
```
┌──────────────────────────────────────┐
│  [←]   页面标题             [操作按钮] │
└──────────────────────────────────────┘
```
- 左侧：返回箭头（`uni.navigateBack()`）
- 中间：页面标题
- 右侧：根据页面不同（分享/更多/编辑等）

### DiaryCard 日记卡片组件

这是最核心的复用组件，出现在日记流、搜索结果、日记日历详情等多处。

```
┌──────────────────────────────────────────┐
│ [情绪emoji]  时间 HH:mm           [···]  │
│                                          │
│ ┌────┐ ┌────┐ ┌────┐                    │（照片区：0-9张网格）
│ │    │ │    │ │    │                    │
│ └────┘ └────┘ └────┘                    │
│                                          │
│ 日记正文预览（最多3行截断）...            │
│                                          │
│ [#标签1] [#标签2] [#标签3]               │（Tag 组，最多显示5个）
│                                          │
│ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  │
│ 🎬 漫画  📤 分享  🎵 BGM  🎙️ 朗读       │（操作栏）
└──────────────────────────────────────────┘
```

| 属性 | 类型 | 说明 |
|---|---|---|
| diaryId | string | 日记 ID |
| emotion | Emotion | 情绪对象（emoji + label + score 1-10） |
| createdAt | timestamp | 创建时间 |
| images | string[] | 图片 URL 数组（0-9张） |
| content | string | 日记正文（卡片中截断 3 行） |
| tags | string[] | 标签数组 |
| style | string | 文风名称（文艺/搞笑/中二等） |
| hasComic | boolean | 是否已生成漫画 |
| hasBGM | boolean | 是否已生成 BGM |

**照片网格规则**：
- 1张：满宽，高度 300rpx，圆角 16rpx
- 2张：并排各 50%，间距 8rpx
- 3张：上1大(2/3宽)+右2小(1/3宽叠两层)
- 4-9张：3列网格，等宽等高

**交互**：
- 点击卡片 → 跳转日记详情页
- 点击照片 → 全屏预览（左右滑动切换）
- 点击标签 → 按标签筛选日记列表
- 点击操作栏按钮 → 跳转对应功能页
- 长按卡片 → 弹出操作菜单（编辑/删除/切换文风/置顶）
- 左滑 → 快捷删除（二次确认）

### 数据加载

- **下拉刷新**：顶部下拉触发，刷新最新日记
- **上拉加载更多**：分页加载，每页 10 条
- **空状态**：新用户无日记时，显示引导卡片「拍张照片，开始你的第一篇日记」+ 空状态插画 `empty-no-diary.png`

# 六、全局交互流程

## 6.1 核心用户旅程

### 旅程 1：拍照写日记（日活核心）

```
用户打开 App
  → 首页日记流
  → 点击中间 ✏️ 按钮
  → ActionSheet: 选择「📸 拍照记录」
  → 打开相机 → 拍照
  → 进入写日记页 (mode=photo)
  → 可选：添加文字/选情绪/加标签
  → 点击 [✨ AI 生成]
  → AI 加载动画 (15秒)
  → 日记生成完成 → 预览
  → 可选：切换文风 / 编辑微调
  → 点击 [完成 ✓]
  → 跳转日记详情页
  → 可选：生成漫画 / 分享卡片 / BGM
```

### 旅程 2：浏览日记 + 生成漫画

```
用户打开 App
  → 首页日记流 → 滚动浏览
  → 点击某条日记卡片
  → 日记详情页
  → 点击 [🎬 生成漫画]
  → 漫画生成页
  → 选择风格（日漫/国漫/...）
  → 点击 [✨ 生成漫画]
  → 加载动画 (30秒)
  → 漫画预览
  → 满意 → [📤 分享] → 分享卡片页（自动选漫画模板）
```

### 旅程 3：AI 对话 + 番茄钟

```
用户打开 App
  → TabBar 切换到「消息」
  → 点击 AI 伙伴入口
  → AI 对话页
  → 用户："今天好焦虑"
  → AI 回复 + [🍅 开个番茄钟学这个]
  → 点击快捷按钮
  → 跳转番茄钟页
  → AI 提问 "做什么？" → 用户回答
  → 开始倒计时
  → 完成 → AI 问结果 → 自动写入日记
```

### 旅程 4：每日碎片 → 文风日记

```
用户白天随手拍照/短文字记录多条碎片
  → 晚上打开 App
  → 发现页 → 文风引擎
  → 查看今日碎片列表
  → 选择文风（武侠/搞笑/文艺...）
  → [✨ 生成今日日记]
  → AI 将碎片聚合成一篇完整日记
  → 预览 → 编辑/换文风 → 保存
```
