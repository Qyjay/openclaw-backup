# 大学生 AI 生活伙伴 — 完整产品构想

> **一句话定义：** 你的大学 AI 伙伴——记录生活、陪伴学习、精准社交、助力成长。
> **技术底座：** 借鉴 OpenClaw Agent 架构的持久化记忆 + 主动式 + A2A 社交引擎。
> **目标路径：** 大学生 → 高中生 → 职场新人 → 全年龄段

---

## 一、为什么大学生是最佳切入点

### 1.1 大学生的独特优势

| 维度 | 为什么适合 |
|---|---|
| **地理聚集** | 同一校园，线下见面零成本 → A2A 匹配后真能约上 |
| **需求密度** | 找搭子/选课/考试/社团，需求真实、高频、场景化 |
| **社恐比例高** | Z 世代社交焦虑普遍，AI 代理社交正中痛点 |
| **信任基础** | 同校同学天然信任 → 冷启动比泛社交容易 10 倍 |
| **数据丰富** | 大学生活事件密集（上课/社团/实习/恋爱/旅行），喂养智能体效率高 |
| **传播力强** | 一个宿舍带一层楼，一个学院带一个学校 |
| **比赛加分** | 南开承办 + 评委多为高校教师 → 大学生产品天然共鸣 |

## 三、模块一：📔 校园日记（流量入口 & 数据引擎）

> 核心定位：零门槛记录 → 喂养 AI → 让 AI 越来越懂你

### 3.1 基础日记功能

| 功能 | 说明 | 大学生场景 |
|---|---|---|
| **拍照日记** | 上传照片 → AI 生成日记 | 食堂新菜、社团活动、自习室、旅行 |
| **语音日记** | 说一段话 → 整理成文 | 通勤路上、睡前碎碎念 |
| **快捷模板** | 一键记录固定场景 | "今天的课/今天吃了啥/今天的心情" |
| **多图叙事** | 多张照片 → 串联成故事 | 一天的旅行、社团活动回顾 |
| **风格切换** | 文艺/搞笑/简洁/中二/吐槽 | 适配不同心情和性格 |

### 3.2 大学生专属日记场景

#### 📸 课表联动日记
- 导入课表（教务系统截图 → AI OCR 识别 → 自动生成课表）
- 日记自动关联当日课程："今天高数课学了泰勒公式，老师讲得很快"
- AI 识别到课程关键词后自动归类到"学习"标签

#### 🗺️ 校园足迹地图
- 记录你常去的地方（图书馆/食堂/操场/奶茶店）
- 生成"我的校园热力图"
- 成就解锁："图书馆常客"（累计打卡 50 天）
- **社交联动**：发现和你去同一间自习室的人

#### 🏠 宿舍故事
- "宿舍日记"专属标签
- 记录室友间的趣事、日常
- 毕业时自动生成"宿舍四年回忆录"

#### 🎭 情绪画布
- AI 从照片+文字自动分析情绪
- 可手动标注（😊😐😢😡😴）
- 情绪日历热力图（一目了然）
- AI 洞察："你周一情绪通常最低——因为有早八。要不要调整一下作息？"
- 考试周情绪追踪："考试周你的平均焦虑指数上升了 40%，但考完后恢复很快"

#### ⏰ AI 早晚安（主动式日记）
- **早安推送**（Cron 08:00）：
  ```
  早上好！☀️ 今天周三，有两节课。
  📋 今日待办：交数据结构作业（截止今晚）
  🍜 食堂推荐：二楼新出了酸菜鱼（你上次说想吃）
  💪 雅思倒计时：还有 43 天
  ```
- **晚安总结**（Cron 22:00）：
  ```
  今日回顾 🌙
  📸 你拍了 4 张照片
  📚 番茄钟完成 6 个（雅思阅读 × 4，编程 × 2）
  😊 今日情绪：7/10（比昨天高）
  💤 建议 23:30 前睡觉，明天有早八
  ```

## 四、模块二：📚 学习伙伴（效率工具 & 长期价值）

> 核心定位：不替你学，但让你学得更聪明

### 4.1 学习管理

| 功能 | 说明 | 技术实现 |
|---|---|---|
| **AI 课堂笔记** | 拍照 PPT/板书 → 结构化笔记 | Vision API + 文本整理 |
| **考试倒计时** | 自动从课表/日记提取考试日期 | Function Calling 提取 |
| **AI 复习计划** | 距考试 N 天 → 自动生成复习排期 | LLM 规划 + 日记学习记录 |
| **番茄钟** | 25min 专注 + 5min 休息 | 本地计时器 + AI 陪伴 |
| **错题拍照** | 拍错题 → AI 解析 → 归档 | Vision 识别 + 知识库存储 |
| **GPA 追踪** | 成绩录入 → 趋势分析 → 选课建议 | 数据分析 + LLM 建议 |

### 4.2 番茄钟 Pro（深度融合 AI）

不是简单计时器，而是：

```
开始前 → AI："这个番茄钟要做什么？" 
        用户："雅思阅读"
        AI："好的，计时开始。上次你做 Passage 1 用了 22 分钟，今天试试 20 分钟以内？"

进行中 → 手机翻转/锁屏 = 专注模式
        AI 不打扰，但默默记录专注时长

结束后 → AI："25 分钟到！完成了吗？"
        用户："做了一篇，对了 9/13"
        AI："正确率 69%，比上次的 62% 进步了！错的主要是 matching 题，
            建议下次重点练这个题型。休息 5 分钟？"

日记自动记录 → "今天番茄钟 6 个，雅思阅读正确率逐步上升"
```

### 4.3 备考专项助手

**大学生高频备考场景：**

| 场景 | AI 能做什么 |
|---|---|
| **期末考试** | 从课堂笔记中提取重点 → 生成思维导图 → 模拟出题 |
| **雅思/托福** | 跟踪每次练习分数 → 分析弱项 → 推荐针对性练习 |
| **考研** | 长期规划拆解 → 每日任务 → 进度追踪 |
| **考公** | 行测专项练习追踪 → 申论 AI 批改 |
| **求职面试** | AI 模拟面试官 → 基于用户画像出针对性问题 |
| **留学申请** | 申请进度管理 → 文书素材从日记中自动提取 |

### 4.4 AI 模拟面试（特色功能）

```
用户选择：求职面试 / 留学面试 / 考研面试
→ AI 读取用户画像（日记中的项目经历、实习经验、兴趣爱好）
→ 生成针对性面试问题
→ 用户语音/文字回答
→ AI 评分 + 改进建议

示例：
AI："看到你简历上写了在 MiniMax 做 AI 售前解决方案实习，
    能具体讲讲你做了什么吗？"
用户回答...
AI："回答不错，但建议用 STAR 法则重新组织：
    Situation → Task → Action → Result
    你提到了做了什么，但结果和数据不够具体。再试一次？"
```

## 五、模块三：👥 校园社交（差异化核心 & A2A 引擎）

> 核心定位：AI 替你迈出社交第一步，你只需要决定要不要见面

### 5.1 为什么大学是 A2A 最佳落地场景

传统社交 APP 的 A2A 面临冷启动难题。但大学场景天然解决了：

```
泛社交 A2A：                     大学校园 A2A：
❌ 用户分散全国                   ✅ 同一校园，步行可达
❌ 需求模糊（交友？恋爱？）        ✅ 需求具体（找雅思搭子/饭搭子/球搭子）
❌ 信任成本高                     ✅ 同校天然信任
❌ 需要百万用户才有匹配密度        ✅ 一个学校几千人就够用
❌ 线下见面成本高                 ✅ "食堂见？" "好的"
```

### 5.2 搭子匹配系统

#### 找搭子的核心流程

```
Step 1: 你发出搭子需求
  "想找人下周六去天津之眼，最好也喜欢拍照"

Step 2: 你的 Agent 后台筛选
  - 扫描校内其他 Agent
  - 匹配条件：周六有空 + 喜欢拍照 + 去过/想去天津之眼
  - 匹配到 3 个候选人

Step 3: Agent to Agent 预沟通
  你的 Agent → 候选 Agent：
  "我的主人周六想去天津之眼拍照，你的主人有兴趣吗？"
  候选 Agent → 你的 Agent：
  "我的主人周六上午有课，下午可以！她最近刚买了新相机想试试。"

Step 4: 双方确认
  推送给你："找到一个搭子！周六下午天津之眼，对方也喜欢摄影。是否建联？"
  你：✅ 同意
  → 交换微信 / 站内聊天

Step 5: 活动后反哺
  回来后上传照片 → 日记自动生成
  → Agent 学到："主人和这个搭子合得来"
  → 下次优先匹配
```

#### 搭子类型清单

| 搭子类型 | 匹配维度 | 场景 |
|---|---|---|
| 🍜 **饭搭子** | 食堂偏好 + 口味 + 空闲时间 | "中午有人想去二食堂吗？" |
| 📚 **自习搭子** | 学习时间 + 自习室偏好 + 安静程度 | "找人一起泡图书馆" |
| 🏃 **运动搭子** | 运动类型 + 水平 + 时间 | "找人打羽毛球双打" |
| ✈️ **旅行搭子** | 目的地 + 时间 + 预算 + 出行风格 | "五一想去长沙，求同行" |
| 📖 **考试搭子** | 同课程 + 同考试 + 互补型 | "高数期末一起复习" |
| 🎮 **游戏搭子** | 同游戏 + 段位 + 时间 | "找人开黑王者" |
| 🎬 **追剧搭子** | 同好剧 + 追番 | "有人一起看 XX 吗？" |
| 💼 **实习搭子** | 同行业 + 同城市 + 互相内推 | "有人也在投北京互联网暑期实习吗？" |
| 🗣️ **语伴** | 语言学习 + 目标 + 水平 | "找英语口语练习搭子" |

### 5.3 校园匿名广场

- **树洞模式**：匿名发布心情/困惑，其他人的 Agent 可以匿名回复鼓励
- **表白墙**：匿名表白/寻人，AI 帮你优化表白措辞
- **失物招领**：AI 自动匹配丢失物品和发现物品的描述
- **二手交易**：闲置物品匹配

### 5.4 社交破冰辅助

大学生最大的社交障碍不是"找不到人"，而是"不知道聊什么"。

```
建联成功后 → AI 分析双方画像 → 生成破冰话题建议：

"你们的共同点：
  🎵 都喜欢听周杰伦
  📚 都在准备雅思（你 6.5，对方 7.0）
  🍜 都标记过食堂二楼的酸菜鱼

破冰建议：
  '听说你雅思考了7分？能分享一下阅读怎么提高的吗？'
  或者：'二楼酸菜鱼你也喜欢！要不一起去吃？'"
```

```
┌─────────────┐         A2A Protocol        ┌─────────────┐
│   用户 A     │                             │   用户 B     │
│             │                              │             │
│  ┌────────┐ │  ① 发起匹配请求              │  ┌────────┐ │
│  │ Agent A │─┼──────────────────────────────┼─▶│ Agent B │ │
```

### 7.2 隐私安全设计

A2A 社交最大的用户顾虑是**隐私**。需要严格分层：

```
┌─ Agent 可共享层（匹配时使用）──────────────────────┐
│  兴趣标签：[摄影, 雅思, 羽毛球, 日本动漫]          │
│  时间偏好：周三下午、周末全天                      │
│  学院：计算机学院（可选是否公开）                  │
│  社交需求：找运动搭子、找自习搭子                  │
└──────────────────────────────────────────────────┘

┌─ Agent 推理层（仅本地分析，不传输）────────────────┐
│  情绪记录、私密日记内容、具体行踪                  │
│  人际关系细节、个人隐私信息                        │
└──────────────────────────────────────────────────┘

┌─ 绝对隐私层（永不离开设备）───────────────────────┐
│  身份证信息、手机号、家庭情况、财务信息            │
│  未授权的任何个人数据                              │
└──────────────────────────────────────────────────┘
```

**端侧模型在这里的价值**：
- 用户画像分析在本地完成 → 数据不上传
- A2A 匹配时只传输"脱敏标签"，不传输原始日记
- 复赛蓝心端侧模型可以做本地推理 → 隐私卖点

## 九、群体衍生路径

### Phase 1：大学生（核心版本）
- 校园日记 + 学习管理 + 搭子社交 + 成长轨迹
- 一所学校验证 → 多所学校推广

### Phase 2：高中生（高考版本）
- **保留：** 日记、情绪追踪、AI 陪伴、运势
- **强化：** 学习管理（高考倒计时、错题本、科目分析）
- **弱化：** 社交功能（高中生时间有限）
- **新增：** 家长端（了解孩子情绪/学习状态，不看具体日记）
- **痛点：** 高中生压力极大，情绪管理需求强烈

### Phase 3：职场新人（毕业第一年）
- **保留：** 日记、情绪追踪、AI 陪伴
- **替换：** 学习管理 → 工作管理（OKR/任务/会议）
- **强化：** 社交功能 → 职场人脉（同行交流、行业搭子）
- **新增：** 职场成长（技能提升、跳槽规划、副业探索）
- **痛点：** 职场迷茫、工作生活平衡、社交圈重建

### Phase 4：全年龄通用版
- 日记作为核心不变
- 模块化功能按年龄/身份自动推荐
- Agent 人设按人生阶段演变
