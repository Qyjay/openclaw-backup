# 【四稿】7 位中奖者，6 个真实故事——他们用 MiniMax 做什么

---

## 开头

今年 1 月底，我们办了一场 Coding Plan 订阅抽奖活动，一共送出去了 7 台 Mac mini。

奖品寄出之后，我们花了点时间挨个联系——想知道这几位「天选之人」平时怎么用 MiniMax，对最新模型感受如何，有没有吐槽。

打通了 6 位。

他们的身份各不相同：有人在南京用 OpenClaw 做新闻聚合和代码编辑，有人在上海白天写代码晚上生成音乐和配音，有人在珠海同时跑好几个大模型做横向对比，有人在法国用 MiniMax 跑 Rust 长任务让模型自主执行几个小时，也有正在探索 Agent 能力边界的实验型用户。

他们的共同点是：都是自己试过之后，觉得好用，才愿意继续用下去。

以下原文呈现，没有加工。

### 钱奎省 · 南京

钱奎省用 OpenClaw 做新闻聚合、行情分析——但最常用的，还是代码编辑。

「整体用下来各个场景都很流畅，代码编辑这块最丝滑，长上下文翻历史代码几乎没有延迟。」他用的是当时的最新模型 M2.5，输入一段旧代码让模型接着写，响应速度让他满意。

不过他也留了一句不客气的建议：「生成质量可以再聪明一点。」

### 沈喜荣 · 上海

沈喜荣是这 6 个人里使用场景最杂的一个。

他用 MiniMax 写代码，也用它生成音乐、配音——三个完全不同的方向，全部用同一个工具搞定。他的总结是：「一站式满足各种创作和开发需求。」

他特别提到 M2.5 升级之后准确率明显提升，国内网络环境下响应很快。多 Agent 并行处理复杂任务的效果也超出预期，「效率比想象中高很多」。

幻觉减少，是他最认可的一点进步。

但他同时也说了一句大实话：「跟 Claude Opus 4.6 比还是有差距，Agent 执行速度希望再快一点。」

### Nils BEGOU · 法国格勒诺布尔

Nils 是这次回访里唯一一位海外用户。

他觉得 MiniMax 最强的地方是计划执行和长时任务。「它能忠实执行计划不中途卡死，如果某件事不 work，会主动找替代方案。」这是他敢把任务放给 MiniMax 跑几个小时的原因。M2.5 的推理速度和工具调用能力让他满意，评价是「outstanding」。

他留了两个期待：希望下一代加入多模态能力，希望模型本体能更大一点。

## 他们眼里的 MiniMax

把这 6 份反馈放在一起看，有几个词出现频率很高。

「快」和「丝滑」——多位用户提到响应速度和长上下文流畅度，国内网络环境无需特殊配置，这一点被反复点名。「稳定」——长逻辑链条幻觉减少，代码重构和 Bug 诊断场景下可靠性提升。「便宜大碗」——性价比优势是真实竞争力，对国内开发者尤其友好。

同时也有两条直接的改进建议：代码生成质量还有提升空间，Agent 执行速度与顶尖模型仍有差距。

这些反馈我们认真看过，也认真对待了。

## 从 6 位用户到 20 + 开源项目

写到这里，我们想借这个机会，跟所有关注 MiniMax 的开发者聊一聊——我们这几个月在做什么，以及我们怎么想这个问题。

做开源这件事，我们有一个朴素的出发点：**让开发者不仅能获取模型能力，还能真正用它构建产品。** 模型权重只是起点。开发者还需要 Agent 框架来跑通执行链路，需要 MCP 工具来接入编辑器和工作流，需要专业的 Skills 来处理 PDF、PPT、Excel，需要学术论文的完整代码和数据来复现和改进。所以我们希望开源的是一整套体系，从模型到工具链到研究。

**模型：从 456B 到 M2.7**

从 2025 年 1 月开源第一个模型到现在，14 个月，我们在 HuggingFace 上发布了 16 个模型权重和 7 个评测数据集，在 GitHub 上发布了 20 多个开源项目。

MiniMax-01 是起点，总参数 456B，训练上下文 100 万 token，推理时最长支持 400 万——这在当时是开源模型中最长的。

M2 系列是一个转折点。从 M2 开始，我们把方向明确转向代码和 Agent。M2 总参数 230B，每个 token 只激活 10B，MoE 架构保证知识容量的同时大幅降低推理成本。在 SWE-bench Verified 上得分 69.4%，当时的开源第一。之后的 108 天里，连续发布 M2.1 和 M2.5，创造了行业最快的迭代速度。

M2.1 将 SWE-bench Verified 推到 74.0%，SWE Multilingual 拿到 72.5%，同期 Claude Sonnet 4.5 是 68%。M2.5 进一步推到 80.2%，在第三方脚手架 Droid 和 OpenCode 上超过 Claude Opus 4.6。推理吞吐达到 100 tokens/s，以这个速度连续工作一小时只需要 1 美金。

M2.7 是我们最新的模型。在 SWE-Pro 上得分 56.22%，VIBE-Pro 上 55.6%，Terminal Bench 2 上 57.0%——均接近 Claude Opus 4.6 的水平。在专业办公领域，GDPval-AA ELO 得分 1500，开源最高。M2.7 还能自行构建复杂的 Agent Harness，驱动自身的强化学习，在内部测试中自主运行超过 100 轮迭代循环，最终在评测集上效果提升 30%。

**工具：让开发者快速用起来**

模型权重开源是第一步。但开发者不只需要一个模型文件——他们需要能直接用来构建产品的工具。

围绕 MCP（Model Context Protocol），我们构建了四个项目。MiniMax-MCP（Python，⭐1.3k）是官方 MCP Server，支持文字转语音、图片生成、视频生成、语音克隆、音乐生成和语音设计，兼容 Claude Desktop、Cursor、Windsurf、OpenAI Agents，uvx minimax-mcp 即可安装。MiniMax-MCP-JS 是功能相同的 JS 实现，支持 stdio / REST / SSE 三种传输模式。MiniMax-Coding-Plan-MCP 面向代码开发场景，提供搜索和图像理解工具。minimax_search 是搜索 MCP Server，支持并行多查询和网页内容理解。

同时，我们也将专业技能包封装成了开源的 Skills（⭐3.4k），覆盖开发（前端全栈、后端架构、Android/iOS 原生）、创意（GLSL Shader、 GIF 动态贴纸）、办公（PDF、PPT、Excel、Word）、多模态（语音合成、音乐生成、视频生成、图像生成）等方向。最近更新了四个 Office Skills——minimax-docx / minimax-xlsx / minimax-pdf / pptx-generator，解决的是同一个核心问题：Agent 生成出来的 Office 文件能不能直接交付。Word 用 .NET OpenXML SDK 而非 python-docx，Excel 绕开 openpyxl 直接操作 XML，PPT 先定义约束体系再做生成，并搭建 Execute → Evaluate → Fix 的自循环进化机制，让 Skill 在自动评测中持续收敛。

OpenRoom 是一个跑在浏览器里的 AI 交互空间。完整的桌面环境，窗口可以拖动、缩放、并排显示，内置 9 个接入 AI Agent 的应用，说一句「播放爵士乐」音乐应用就开始播放，数据全部存在本地 IndexedDB，没有后端。更重要的是，我们为开发者封装了 Vibe Workflow：在 Claude Code 里描述你想要什么应用，系统自动走完需求分析 → 架构设计 → 代码生成 → 资源生成 → 项目集成的全流程，生成一个可以直接在 OpenRoom 桌面上运行的完整应用。「这个里面的代码大部分也是 AI 写的。」

此外还有 MiniMax-Provider-Verifier。开源模型交给第三方部署商接入之后，推理框架差异、量化精度选择、解码参数配置都可能导致模型行为和官方部署不一致。这个工具设计了六个维度的评测指标——查询成功率、工具调用触发一致性、Schema 准确率、响应完整性、语言跟随率——跑 10 轮取均值和官方部署做对比，目前已覆盖 M2、M2.1、M2.5 三代模型在十多家部署商上的测试结果，全部公开在 README 里。

**研究：代码、数据、模型，一个都不能少**

我们的学术论文不只发 PDF。每一篇都把代码框架、数据集和训练好的模型一起开源，确保可复现。

SynLogic（NeurIPS 2025）是逻辑推理的数据合成框架，覆盖 35 种任务类型，每个样本都有基于规则的验证器，可直接用于 RL 训练。在 BBEH 基准上，基于 SynLogic 训练的模型超过 DeepSeek-R1-Distill-Qwen-32B 六个百分点。完整框架、SynLogic 数据集（49.3k 样本）和三个训练好的模型全部开源。

V-Triune 是统一的视觉 RL 系统，训练出来的 Orsta 模型在 MEGA-Bench Core 上最高提升 14.1%。VTP（Visual Tokenizer Pre-training）解决了视觉 tokenizer 重建损失和生成质量之间的矛盾——把预训练算力放大 10 倍，FID 改善 65.8%，而传统 autoencoder 在 1/10 算力就停滞了。最终模型在 ImageNet 上重建 rFID 0.36，零样本分类准确率 78.2%，超过 VILA-U 和 UniTok 等统一 tokenizer。

MiniMax-Speech 做到了真正的 zero-shot 语音克隆——不需要参考音频的转录文本，只需要一段参考音频。Speech-02-HD 在 Artificial Arena 公开 TTS 排行榜上拿到第一名，ELO 评分超过 OpenAI 和 ElevenLabs。由于 Speaker Encoder 表征足够鲁棒，模型不改基座就能用 LoRA 控制情绪、Text-to-Voice 直接生成音色、用少量数据做专业级语音克隆。

## 说在最后

回到开头那 6 位用户。

他们的使用场景各不相同——有人在南京写代码，有人在珠海同时跑多个模型横评，有人在法国让 AI 自主执行任务几个小时。他们的反馈里有好评，也有直接的改进建议。

这些反馈我们都认真记下了，也在持续迭代。

两个月前有用户说「希望跟 Claude Opus 4.6 的差距再小一点」。今天，M2.7 在 SWE-Pro、VIBE-Pro、Terminal Bench 2 上的得分都已经接近 Opus 4.6 的水平，这个差距在缩小。

也有用户说「希望加入多模态」。今天，Token Plan 已经支持 Hailuo 视频模型、Speech 语音模型、Music 音乐模型和 Image 图像生成模型，一个 Key 全部搞定。

还有用户在探索 Agent 能力的边界。今天，M2.7 已经能够自行构建 Agent Harness，驱动自身强化学习，在 100 多轮的自主迭代中让效果提升 30%。

模型在变强，工具在完善，开源项目在持续增加。如果你在这 6 位用户的反馈里看到了自己的影子，欢迎来试试。
