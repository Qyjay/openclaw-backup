AIHost
一句话定位 / 项目简介
AIHost，Your AI Event Host——AI 时代就用 AI 主持人。
AIHost 是一款面向线上与线下活动场景的 AI 主持人产品，覆盖主持全流程：从主持稿生成、播报音频生成，到现场自动口播与实时观众互动。产品结合 ASR 实时追踪嘉宾分享进度，自动提取核心观点，生成个性化过渡口播，尽可能减少 human in the loop，让活动主持这件事真正进入 AI 时代。

二、项目亮点
核心功能 / 创新点
1. 主持稿智能生成：上传活动议程图片或文档，多模态模型自动理解流程并生成完整主持稿；用户可通过文字或语音与模型对话进行实时修改优化。
2. AI 驱动的动态口播（Enhance Line）：在嘉宾演讲过程中，系统持续记录 ASR 转录内容，引用嘉宾实际发言生成个性化过渡词——而非固定台词，让主持更生动、更有即时感。
3. 自动主持：流式 ASR 实时监听、自动触发口播
开发中遇到的关键技术难点
- 低延迟播报：【主持】要求延迟要很低，嘉宾说完到 AI开始主持之间不能有明显停顿，通过调用小模型和合理的做chunk降低了延迟
- 台词拆分 Prompt 设计：主持稿混杂流程说明与真实台词，需要模型精准识别并只提取需播报的部分，不改动原文——这套 prompt 经过多轮迭代才稳定下来。
解决了什么真实痛点 / 创业初衷
活动主持是一件被严重低估成本的事：赛前写稿半天到一天、活动中连续高强度工作数小时、任何临时变动都靠主持人现场扛。而在 AI 时代，尤其是在一场 OpenClaw 主题的黑客松上，这件事本身就不应该还需要人来做。AIHost 的初衷很简单：让活动主持这件事，真正进入 AI 时代。

 二、项目亮点
 - 核心功能 / 创新点：ClawSight 的核心不是简单做一个“新闻聚合器”，而是希望把“信息获取—事实核查—研究组织—历史类比—市场传导分析—结果输出”做成一条自动运行的 AI 研究流水线。
 第一，多 Agent 协作。
 我们把复杂研究任务拆成多个角色，每个 Agent 只负责自己最擅长的部分，比如信息采集、事实验证、事件框架整理、历史案例研究、市场影响分析等，最后再由主 Agent 汇总输出。这样比单模型一次性回答更稳定，也更容易持续优化。
 第二，关注“事件传导”而不是只做信息摘要。
 很多产品停留在“告诉你发生了什么”，但 ClawSight 更想回答“这件事为什么重要，会影响谁，会怎么传导，下一步应该关注什么”。
 第三，可持续迭代。
 每个 Agent 在运行中都会沉淀自己的判断样本和训练数据，后续可以持续校准和增强，让系统不是一次性回答，而是越来越像一个真正会成长的研究系统。
 - 开发中遇到的关键技术难点
 最大的难点是怎么让多个 Agent 协作后仍然保持结果稳定、上下文一致、任务边界清晰。
 比如：
 - 不同 Agent 拿到的信息上下文怎么裁剪，既保证足够完整，又不浪费 token
 - 多个 Agent 输出的结果怎么统一结构，避免重复、冲突或者失焦
 - 如何让系统在信息不完整、信源相互矛盾时仍然能给出相对可靠的判断
 - 如何把“研究逻辑”变成可执行、可复用、可扩展的工程化流程
