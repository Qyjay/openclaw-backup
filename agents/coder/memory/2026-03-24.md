## 🔴 教训：SVG data URI 中的颜色双重编码（2026-03-24 16:33）

### 问题
DoodleIcon 组件在 UniApp App 模式下图标不显示。带 `#` 的颜色（如 `#E8855A`）的图标全部不渲染，纯英文颜色名（如 `red`、`blue`）的图标正常显示。

### 根因
`escapeColor()` 函数把 `#` 预转义为 `%23`，然后 `encodeURIComponent()` 又把 `%` 编码为 `%25`，导致最终颜色值变成 `%2523E8855A`（无效颜色）。

```
原始: #E8855A
escapeColor: %23E8855A
encodeURIComponent: %2523E8855A  ← 双重编码，废了
```

### 正确做法
直接用原始颜色值，让 `encodeURIComponent` 一次性处理所有特殊字符：
```js
// ❌ 错误：手动预转义 + encodeURIComponent = 双重编码
stroke="${escapeColor(props.color)}"  →  encodeURIComponent(svg)

// ✅ 正确：直接用原始值
stroke="${props.color}"  →  encodeURIComponent(svg)
```

### 铁律
**永远不要在 `encodeURIComponent` 之前手动转义字符。** 让 `encodeURIComponent` 统一处理，不要画蛇添足。

### 延伸
UniApp App 模式不支持 inline SVG（`<svg>` 标签直接写在 template 里不渲染）。必须用 `<image>` + SVG data URI 方案：
```js
const svg = `<svg xmlns="...">${paths}</svg>`
return `data:image/svg+xml,${encodeURIComponent(svg)}`
```
用 `<image :src="dataUri">` 渲染，所有平台兼容。

## 🔴 教训：UniApp App 模式 box-sizing 全局覆盖（2026-03-24 17:02）

### 问题
所有页面内容往右下角溢出屏幕边缘。

### 根因
App.vue 里 `box-sizing: border-box` 只设了 `view, text, input, textarea`，但 UniApp 的 `scroll-view`、`swiper`、`image` 等是独立组件标签，没被覆盖到。`scroll-view` 仍用 `content-box`，padding 被加到宽高之外，导致溢出。

### 修复
```css
*, *::before, *::after {
  box-sizing: border-box;
}
```
用通配符全局覆盖，确保所有 UniApp 组件都用 `border-box`。

### 铁律
**UniApp 项目必须用 `*` 通配符设置 `box-sizing: border-box`。** 不要只列 `view, text` 等标签——UniApp 有大量自定义组件标签（scroll-view, swiper, image 等）会被遗漏。

## 🔴 教训：UniApp App 模式不支持 inline SVG（2026-03-24 16:22）

### 问题
`<svg>` 标签直接写在 Vue template 里，在 App 模式（Android WebView）中完全不渲染。H5 模式正常。

### 修复
改用 `<image>` + SVG data URI 方案：
```js
const svg = `<svg xmlns="...">${paths}</svg>`

// <image :src="dataUri" mode="aspectFit" />
```

### 铁律
**UniApp App 模式禁用 inline SVG。** 必须用 `<image>` + data URI 或图片文件。

## 🔴 教训：UniApp App 模式 `<transition>` 不可靠（2026-03-24 17:40）

### 问题
TabBar 中间按钮弹出的 ActionSheet 点取消无法关闭；SideDrawer 侧边栏打开后无法点击收回。

### 根因
Vue 的 `<transition>` 组件在 UniApp App 模式（Android WebView）下行为不可靠。`<transition name="sheet">` 没有定义对应的 CSS transition 类（`sheet-leave-active` 等），导致 leave 动画永远不结束，`v-if` 的元素无法从 DOM 中移除，overlay 始终存在阻挡点击。

### 修复
直接去掉 `<transition>` 包裹，用 `v-if` 裸控制显示/隐藏。

### 铁律
**UniApp App 模式禁用 `<transition>`。** 如需动画效果，用 CSS class 切换 + `v-if`/`v-show` 直接控制。

## 🔴 教训：scroll-view padding-bottom 会形成可见空白区域（2026-03-24 17:52）

### 问题
「我的」页面 TabBar 上方出现一块与背景同色的纯色长方形遮挡。

### 根因
scroll-view 设了 `padding-bottom: 180rpx` 给 TabBar 留空间，但这个 padding 区域本身就是一个可见的空白块（继承了页面背景色 `#FDF8F3`）。180rpx 过大，超出了 TabBar 的实际覆盖范围（120rpx），多余的 60rpx 成为可见空白。

### 修复
`padding-bottom` 改为 `130rpx`（略大于 TabBar 高度 120rpx，留 10rpx 余量）。

### 铁律
**有 TabBar 的页面，scroll-view 的 padding-bottom 应略大于 TabBar 实际高度（120rpx + 10rpx 余量 = 130rpx），不要过大。**

## 🔴 教训：TabBar wrapper 不能用 position: relative（2026-03-24 17:49）

### 问题
TabBar 组件的 `.tabbar-wrapper` 用了 `position: relative`，虽然内部 `.tabbar` 是 `position: fixed`，但 wrapper 本身在文档流中仍占据空间，形成空白遮挡。

### 修复
`.tabbar-wrapper` 也改为 `position: fixed; bottom: 0`，用 `pointer-events: none` + 子元素 `pointer-events: auto` 避免阻挡点击。`.tabbar` 改为 `position: relative`（相对于 fixed 的 wrapper）。
