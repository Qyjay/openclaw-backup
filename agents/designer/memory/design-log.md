# 配色修订记录 — v1.0 → v1.1

**日期**：2026-03-23  
**版本**：Design System v1.1  
**执行**：Iris 🎨（视觉总监）  
**指令来源**：Master Kylin

---

## 变更摘要

**方向**：整体配色从「暖紫 + 珊瑚」转为「暖杏橙 + 奶油白」暖色调系统。  
**感觉目标**：像一杯温暖的奶茶——柔和、舒适、不刺眼。

---

## 配色变更明细

### 主色：暖紫 → 暖杏橙

| | 旧值 | 新值 |
|---|---|---|
| 主色 | `#7B5EA7` 暖紫 | `#E8855A` 暖杏橙 |
| 深色 | `#5B3A8A` | `#C05C30` |
| 浅色 | `#9E7DC4` | `#F0A882` |
| 淡色 | `#C4AEDF` | `#F7CDB5` |
| 底色 | `#EDE7F6` | `#FDF0E8` |

**决策理由**：Master 要求淡色系+暖色调，以米色、白色、橙色为主。暖杏橙 `#E8855A` 是柔和的暖橙（非鲜艳橘红），饱和度适中，符合"奶茶"气质。与原紫色相比，更贴近大学生日记类产品的温暖感。

### 辅助色：珊瑚橙 → 淡珊瑚粉

| | 旧值 | 新值 |
|---|---|---|
| 辅助色 | `#FF7E6B` 珊瑚橙 | `#F2B49B` 淡珊瑚粉 |
| 浅色 | `#FFA99A` | `#F7CEB9` |
| 底色 | `#FFF0EE` | `#FEF3EE` |

**决策理由**：旧珊瑚橙偏鲜艳，在全暖色系下对比不够柔和。新淡珊瑚粉降低饱和度，与主色暖杏橙形成同色系渐进关系，整体更温润。

### 页面底色：暖白 → 奶油白

| | 旧值 | 新值 |
|---|---|---|
| N50 页面底 | `#FAFAF8` | `#FDF8F3` |
| N100 输入框底 | `#F3F3F0` | `#F5EDE4` |
| N200 分割线 | `#E8E8E4` | `#EAE0D6` |

**决策理由**：原 `#FAFAF8` 已是暖白，但微暖程度不够。新 `#FDF8F3` 带更明显的米黄调，视觉上像奶油纸，实现"奶茶"质感。

### 中性灰：冷灰 → 暖灰（偏棕黄）

| 级别 | 旧值 | 新值 |
|------|------|------|
| N300 | `#D0D0CA` | `#D4C4B8` |
| N400 | `#A8A8A2` | `#AE9D92` |
| N500 | `#808078` | `#857268` |
| N700 | `#3D3D38` | `#4A3628` |
| N900 | `#1A1A2E`（带蓝） | `#2C1F14`（纯暖棕） |

**决策理由**：彻底消除冷灰成分。N900 标题色从带微蓝的深色改为纯暖棕 `#2C1F14`，与整体暖色系统一。暖灰在奶油底色上的搭配更和谐。

### 模块专属色：彩色 → 暖色系

| 模块 | 旧值 | 新值 |
|------|------|------|
| 📔 日记 | `#E8A87C` 暖橙 | `#E8855A` 暖杏橙（与主色统一） |
| 📚 学习 | `#5BA4CF` 宁静蓝 | `#D4956A` 暖棕杏 |
| 👥 社交 | `#6EC6A0` 清新绿 | `#C8A882` 浅暖驼 |
| 🌱 成长 | `#F0C26E` 温暖金 | `#E6B870` 琥珀金（保持，微调） |
| 🔮 运势 | `#B47FEB` 神秘紫 | `#D4956A` 暖焦糖 |

**决策理由**：原模块色冷暖混杂（蓝/绿破坏整体感）。全部调整为暖色谱：橙-棕-驼-金-焦糖，形成同色系但有区分度的模块主题。

### 功能色：微调协调暖调

| 语义 | 旧值 | 新值 | 变化 |
|------|------|------|------|
| 成功 | `#4CAF82` | `#5BAF85` | 微调偏绿茶感 |
| 警告 | `#F5A623` | `#E8962A` | 微调偏琥珀感 |
| 错误 | `#F55B5B` | `#D95C4A` | 从冷橙红→暖砖红 |
| 信息 | `#64B5F6` | `#7BB8D4` | 降饱和，偏灰蓝 |

**决策理由**：功能色语义必须保留，但将饱和度和色温微调，避免在暖色调整体中显得突兀。错误色变化最大：从纯橙红改为暖砖红，减少刺眼感。

# 设计决策记录 — AIGC 比赛项目设计系统

**日期**：2026-03-23  
**版本**：Design System v1.0  
**项目**：大学生 AI 生活伙伴

---

## 关键设计决策

### 1. 主色选择：暖紫 #7B5EA7（非蓝色）

**决策**：放弃常见的蓝色（科技感），选择暖紫作为主色。  
**理由**：
- 目标用户 18-24 岁大学生，产品调性是"贴心的 AI 朋友"，不是"冷冰冰的工具"
- 蓝色在同类 AI 产品（ChatGPT、Claude）泛滥，缺乏差异化
- 暖紫兼具科技感（紫色的神秘智慧）+ 亲近感（暖调），恰好契合"智慧伙伴"人设
- 与辅助色珊瑚橙 #FF7E6B 搭配形成「紫+橙」互补色方案，活力但不刺眼
- 参照：Notion（灰）过于严肃；Soul（蓝紫）视觉接近但更冷；我们的暖紫更年轻

### 2. 页面底色：米白 #FAFAF8（非纯白）

**决策**：使用略带暖调的米白而非纯白 #FFFFFF 作为页面背景。  
**理由**：
- 纯白屏幕在大量阅读时刺眼（日记时间线场景高频阅读）
- 微暖底色呼应整体温暖调性，与卡片白色 #FFFFFF 形成层次区分
- Apple Notes、Day One 等优秀日记 App 均使用类似策略

### 3. 5个模块专属色策略

**决策**：5个模块（日记/学习/社交/成长/运势）各有独立主题色，但均为柔和饱和度。  
**理由**：
- 用户需要在视觉上快速区分所在模块（减少认知负担）
- 高饱和度颜色拼在一起会"撞色"，统一降低饱和度保持整体和谐
- 日记=暖橙（温暖记忆）/ 学习=宁静蓝（专注）/ 社交=清新绿（连接）/ 成长=温暖金（收获）/ 运势=神秘紫（玄学感）
- 每种颜色都有 10% 透明度背景版本，用于卡片底色不抢主色风头

### 4. 品牌阴影带紫色调

**决策**：Level 2/3 阴影使用 `rgba(123, 94, 167, 0.10~0.18)` 而非纯黑阴影。  
**理由**：
- 纯黑阴影 `rgba(0,0,0,0.1)` 在白卡上显得"脏"
- 品牌紫色调阴影让卡片悬浮感更有品牌一致性，整体更精致
- 参照 Phantom Wallet 的 neon 阴影策略：阴影与品牌色挂钩

### 5. NavBar / TabBar 毛玻璃效果

**决策**：导航栏和 TabBar 使用 `backdrop-filter: blur(20px)` + 92-95% 透明度背景。  
**理由**：
- 大学生熟悉 iOS 设计语言，毛玻璃是现代 App 的质感标志
- 日记时间线页面内容滚动时，导航栏与内容有视觉分层，避免混乱
- UniApp Android 端需要测试兼容性：Android 12+ 原生支持；低版本 fallback 为纯色背景

### 6. TabBar 首页中间凸起设计

**决策**：首页 Tab（🏠）设计为圆形凸起样式，高于其他 Tab。  
**理由**：
- 首页是全局门户，视觉权重需要最高
- 凸起形态制造惊喜感，符合 18-24 岁用户审美偏好
- 避免 5 Tab 等权视觉上的单调
- 参照：Instagram、TikTok 的中间 + 按钮方案

### 7. DIN Alternate 等宽数字字体

**决策**：计时器、统计数字、成就数字使用 DIN Alternate / SF Mono。  
**理由**：
- 普通 PingFang SC 数字非等宽，计时器跳动时会产生抖动
- DIN Alternate 是 Apple 系统内置字体，Android 可降级到 Roboto Condensed
- 等宽数字在番茄钟倒计时场景下视觉更稳定，专业感更强

### 8. 情绪选择器 emoji 方案

**决策**：使用系统 emoji（而非自定义图标）做情绪选择器。  
**理由**：
- emoji 天然有亲切感，目标用户高度熟悉
- 减少美术资源依赖，节省开发时间
- 系统 emoji 在不同 Android 版本有差异，备选：使用 Twemoji CDN 统一风格
