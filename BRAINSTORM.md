## 🟢 快速模式：热点扫描 + 批量创意（10 分钟）

### Step 1: 抓取当下热点

```
Silvana 并行执行：
├── Google Trends：中国 + 全球 AI 相关热搜
├── Web Search：近 7 天 AI 应用/产品新闻
├── Web Search：Product Hunt 最新 AI 产品
├── Web Search：GitHub trending AI 项目
└── Web Search：OpenClaw / Agent 生态最新动态
```

### Step 2: 提取信号

从热点数据中提取：
- **上升趋势**：哪些关键词在快速增长？
- **未被满足的需求**：用户在抱怨什么？
- **技术新能力**：什么以前做不到现在能做了？
- **空白市场**：哪些领域还没有好的 AI 解决方案？

### Step 3: 批量生成 + 评分

使用 business-ideas 的 5 套框架轮换生成：

| 框架 | 思路 |
|------|------|
| **痛点挖掘** | 用户最大的痛点是什么？AI 能怎么解决？ |
| **趋势骑乘** | 哪个正在爆发的趋势可以搭上？ |
| **技术套利** | 什么技术刚成熟但还没被充分应用？ |
| **受众优先** | 特定人群（开发者/学生/创作者）最需要什么？ |
| **商业模式混搭** | 把 X 的商业模式应用到 Y 领域会怎样？ |

每个创意 5 维评分（1-10）：

| 维度 | 核心问题 |
|------|---------|
| 📈 Market | 有多少人需要这个？需求是否已被验证？ |
| ⏰ Timing | 为什么是现在？什么变了？ |
| 🏰 Moat | 能防住竞争对手吗？ |
| 🎯 Founder-fit | Master（软件工程 + MiniMax AI 经验）有优势吗？ |
| 🚀 Simplicity | 30 天内能做出 MVP 吗？ |

**Viability = 平均分。7+ 为高潜力。**

### Step 4: 输出排名表

```markdown
| # | 创意名 | 一句话 | 市场 | 时机 | 护城河 | 匹配 | 简洁 | 总分 |
|---|--------|--------|------|------|--------|------|------|------|
| 1 | XXX    | ...    | 9    | 8    | 7      | 9    | 8    | 8.2  |
| 2 | YYY    | ...    | 8    | 9    | 6      | 7    | 9    | 7.8  |
```

保存到 `~/business-ideas/ideas.md`

## 🟡 深度模式：全链路研究（30-60 分钟）

### Phase 1: 趋势研究（Vibe Research）

Silvana spawn 研究子 agent，执行完整研究管线：

```
研究任务：[具体方向，如"AI Agent 个人助手赛道"]

1. 识别空白：该领域什么问题还没被解决？
2. 文献综合：
   - 最新论文 / 博客 / 产品发布
   - 竞品分析（已有的解决方案）
   - 技术成熟度评估
3. 假设生成：提出 3-5 个可测试的产品假设
4. 交叉验证：用多个来源验证假设
5. 综合报告：发现 + 机会 + 风险
```

### Phase 2: 跨域灵感（Cross-Pollination Engine）

```
问题：[从 Phase 1 发现的核心需求]
核心任务：[去掉行业背景后的本质需求]

FROM [游戏行业]：
  他们怎么解决的：XP系统 + 成就
  核心原则：进度可视化 + 即时反馈
  应用到我们：...

FROM [金融行业]：
  他们怎么解决的：风控模型 + 自动审批
  核心原则：规则引擎 + 置信度评分
  应用到我们：...

SYNTHESIS: [综合方案]
NEXT STEP: [具体行动]
```

### Phase 3: 创意精炼 + 深度评估

对 Phase 1-2 中最有潜力的 2-3 个方向做深度分析：

```
每个方向输出：
├── 商业模式画布（用户/价值/渠道/收入）
├── 3 大风险
├── 验证实验（最快怎么验证这个想法？）
├── MVP 范围（最小可用产品需要什么功能？）
├── 预估开发时间
└── 与 Master 能力的匹配度分析
```

保存到 `~/business-ideas/favorites.md`

## 🟠 评估模式：单点子深潜（15 分钟）

Master 已有想法时使用。

### Step 1: 5 维评分

对想法做 business-ideas 的标准 5 维打分。

### Step 2: 可行性验证

```
├── 竞品搜索：已有哪些类似产品？他们的问题是什么？
├── 技术可行性：现有 API/模型能支撑吗？
├── 市场验证：Google Trends 热度 + 社区讨论量
└── MVP 路径：最小化到什么程度还有价值？
```

### Step 3: 给出裁决

- 🟢 **STRONG YES** — 机会清晰，立刻开干
- 🟡 **CONDITIONAL YES** — 有前景但需要先验证某个假设
- 🟠 **PIVOT RECOMMENDED** — 核心洞察好但执行方向需调整
- 🔴 **PASS** — 红旗太多，不值得投入
