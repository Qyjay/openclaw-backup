# Chronicle — 编年史

Master 与 Agent 团队的历史记录系统。记录从诞生到未来的所有关键节点。

## 编年史文件

- **主文件**: `CHRONICLE.md`（workspace 根目录）
- 按时间倒序排列（最新在最前）
- 每条记录包含时间戳、事件类型、内容

## 记录时机

以下事件发生时，**必须**写入编年史：

| 事件类型 | 标签 | 示例 |
|---------|------|------|
| 团队诞生/Agent 创建 | `🌅 GENESIS` | 系统首次上线、新 Servant 创建 |
| Agent 删除/退役 | `🌑 FAREWELL` | Servant 被移除 |
| Skill 变更 | `🧩 SKILL` | 安装、卸载、更新 skill |
| Master 命令 | `⚔️ ORDER` | 重要任务指派、战略决策 |
| 任务完成 | `✅ DONE` | Servant 汇报完成、交付成果 |
| 团队变更 | `👥 TEAM` | 架构调整、角色变更、路由变更 |
| 系统变更 | `⚙️ SYSTEM` | 配置变更、渠道接入、模型切换 |
| 里程碑 | `🏆 MILESTONE` | 重要成就、阶段性成果 |
| 备忘 | `📝 NOTE` | Seneschal 认为值得记录的事 |

## 记录格式

```markdown
### [YYYY-MM-DD HH:mm] 标签 — 标题

内容描述。简洁但信息完整。

- 关键细节 1
- 关键细节 2
```

## 规则

1. **简洁有力** — 每条记录 2-5 行，不写废话
2. **事实优先** — 记录客观事实，不记录推测
3. **倒序排列** — 最新事件在 CHRONICLE.md 最上方（`---` 分隔线之后）
4. **主动记录** — 不需要 Master 提醒，该记的自己记
5. **可搜索** — 标签统一，方便日后回顾
6. **不记密码/密钥** — 安全信息不入编年史

## 查询

Master 可以说：
- "查看编年史" → 读取 CHRONICLE.md
- "最近发生了什么" → 显示最近 5-10 条
- "查所有 SKILL 相关记录" → 按标签过滤
- "X月发生了什么" → 按时间过滤
